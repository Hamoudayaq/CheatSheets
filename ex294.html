ansible-navigator Configuration

This YAML snippet configures the behavior of ansible-navigator for automation workflows using execution environments.

Example Configuration

---
ansible-navigator:
  execution-environment:
    image: utility.lab.example.com/ee-supported-rhel8:latest
    pull:
      policy: missing
  playbook-artifact:
    enable: false
  mode: stdout

üîß Configuration Breakdown

execution-environment

Defines settings for the container-based execution environment that ansible-navigator uses.

  • image:
    Specifies the container image to use.
    Example: utility.lab.example.com/ee-supported-rhel8:latest
  • pull.policy:
    Determines when to pull the image:
    • missing: Only pull if the image does not already exist locally.

playbook-artifact

Manages JSON artifact generation for playbook runs.

  • enable: false
    Disables playbook artifact generation. This is useful when playbooks require interactive input (e.g., prompting for passwords).

mode

Specifies how output is displayed when running ansible-navigator.

  • Options:
    • interactive: Launches a TUI (Text User Interface)
    • stdout: Outputs results to the terminal (suitable for scripting)
  • üîÅ Use -m stdout or -m interactive to override temporarily.

‚úÖ Best Practice Tip

Use mode: stdout for CI/CD pipelines or automation scripts, and interactive for debugging or manual playbook runs.
