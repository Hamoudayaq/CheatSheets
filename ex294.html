---
ansible-navigator:
  execution-environment: 
    image: utility.lab.example.com/ee-supported-rhel8:latest 
    pull:
      policy: missing 
  playbook-artifact: 
    enable: false 
  mode: stdout

The execution-environment section configures settings for the automation execution environment that the ansible-navigator command uses.

The image key defines the container image name to use for the automation execution environment.

The policy key nested below the pull section states to only pull the container image if it does not already exist on the local machine.

The playbook-artifact section configures settings for the JSON files that Ansible generates every time you run a playbook. Each generated JSON file records information about a specific playbook run. You can use these files to review the results of a playbook run or to troubleshoot playbook issues.

The enable key nested below the playbook-artifact section disables generating playbook artifacts when using the ansible-navigator run command. Playbook artifacts must be disabled when you require a prompt for a password when running a playbook. You can temporarily override this setting from the command line with the --pae option.

The mode key defines the output mode for the ansible-navigator command. The value for this key should be set to either interactive (the default) or stdout. You can temporarily override this setting from the command line with the -m option.
